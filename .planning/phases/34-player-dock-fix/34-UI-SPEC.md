## Layout Contract (Phase-Specific)

This phase is a layout fix, not a feature build. The visual contract is spatial:

### Positioning Rules

| Rule | Specification | Source |
|------|---------------|--------|
| Player-to-tab-bar gap | 0pt -- flush, no gap, no overlap | D-01 |
| Player top shadow | `shadow(color: .black.opacity(0.1), radius: 4, y: -2)` -- retained | D-02 |
| Layout mechanism | `.safeAreaInset(edge: .bottom)` on TabView | D-03 |
| Player-hidden behavior | Content reclaims full space to tab bar, animated via `BSAnimation.smooth` | D-04 |

### Component Dimensions

| Component | Height | Source |
|-----------|--------|--------|
| MiniPlayerView | ~64pt (content-driven: `Spacing.sm` top/bottom padding + inner content) | `ComponentSize.miniPlayerHeight` reference |
| Tab bar | System default (~49pt + safe area) | UIKit standard |

### Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Tab bar taps with player visible | All tab items remain fully tappable -- no z-order obstruction |
| Scroll content behind player | Content insets correctly via `.safeAreaInset` -- no clipping behind player |
| Player show transition | `.opacity` transition with `BSAnimation.smooth` timing |
| Player hide transition | Same -- content reflows downward to fill space |

### Screen Size Contract

| Device Class | Verification |
|-------------|--------------|
| iPhone SE (3rd gen) | Player + tab bar fit without compressing scrollable content to unusable height |
| iPhone 15 Pro | Standard layout, home indicator safe area respected |
| iPhone 15 Pro Max | Same as above, larger viewport |
