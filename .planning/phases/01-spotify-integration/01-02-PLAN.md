    **Replace `SpotifyPlayerService.swift` stub** with full implementation:
    - `@Observable` class, singleton, inherits from NSObject (needed for SPTAppRemote delegates)
    - Conforms to `SPTAppRemoteDelegate` and `SPTAppRemotePlayerStateDelegate`
    - Properties: `isConnected: Bool`, `currentTrack: SpotifyTrack?` (our model, not SPTAppRemoteTrack -- convert in delegate), `isPaused: Bool`, `currentTrackImageURL: String?`
    - Private: lazy `appRemote: SPTAppRemote` configured with clientID and redirectURL (same as auth service -- extract constants to a shared `SpotifyConfig` struct or file if needed)
    - `func connect()`:
      - Guard KeychainManager has access token
      - Set `appRemote.connectionParameters.accessToken`
      - Call `appRemote.connect()`
      - Must be called on main thread
    - `func disconnect()`:
      - If `appRemote.isConnected`, call `appRemote.disconnect()`
    - `func play(uri: String)`:
      - `appRemote.playerAPI?.play(uri, callback: defaultCallback)`
    - `func resume()`:
      - `appRemote.playerAPI?.resume(defaultCallback)`
    - `func pause()`:
      - `appRemote.playerAPI?.pause(defaultCallback)`
    - `func togglePlayPause()`:
      - If isPaused, resume. Else pause.
    - `func skipNext()`:
      - `appRemote.playerAPI?.skip(toNext: defaultCallback)`
    - SPTAppRemoteDelegate implementation:
      - `appRemoteDidEstablishConnection`: set `isConnected = true`, subscribe to player state, set player state delegate
      - `didFailConnectionAttemptWithError`: set `isConnected = false`, log error
      - `didDisconnectWithError`: set `isConnected = false`
    - SPTAppRemotePlayerStateDelegate:
      - `playerStateDidChange`: update `isPaused` from `playerState.isPaused`, convert `playerState.track` (SPTAppRemoteTrack) to our `SpotifyTrack` model (extract name, uri, artist name, album name from the SPTAppRemoteTrack properties). Note: SPTAppRemoteTrack has `.name`, `.uri`, `.artist.name`, `.album.name` but no `id` or `duration_ms` -- store what's available, leave others as defaults.
      - Also call `AudioSessionService.shared.updateNowPlayingInfo(title:artist:duration:)` when track changes

    **Create `SpotifyAPIService.swift`:**
    - Singleton `static let shared`
    - Private `baseURL = "https://api.spotify.com/v1"`
    - `func fetchPlaylists(offset: Int = 0, limit: Int = 50) async throws -> PaginatedResponse<SpotifyPlaylist>`:
      - GET `/me/playlists?limit=\(limit)&offset=\(offset)`
    - `func fetchPlaylistTracks(playlistID: String, offset: Int = 0, limit: Int = 100) async throws -> PaginatedResponse<PlaylistTrackItem>`:
      - GET `/playlists/\(playlistID)/tracks?limit=\(limit)&offset=\(offset)`
    - `func fetchCurrentUserProfile() async throws -> SpotifyUser`:
      - GET `/me`
    - Private `func authenticatedRequest<T: Decodable>(url: URL) async throws -> T`:
      - Get token from KeychainManager; throw `.notAuthenticated` if nil
      - Create URLRequest with Bearer token header
      - Execute with URLSession.shared.data(for:)
      - Check HTTP status: 401 -> throw `.tokenExpired`, other 4xx/5xx -> throw `.apiError`
      - Decode and return
    - Use `JSONDecoder()` with default key decoding (Spotify uses snake_case, match in model with exact property names or CodingKeys)

    **Create `SpotifyAPIServiceTests.swift`:**
    - `testPlaylistDecoding`: Decode the playlist JSON fixture from MockSpotifyResponses, verify fields (name, track count, image URL)
    - `testPlaylistTrackDecoding`: Decode playlist tracks fixture, verify track name, artist, album, URI
    - `testPaginatedResponseDecoding`: Verify total, offset, limit, next fields parse correctly
    - `testEmptyPlaylistResponse`: Decode empty items array, verify items.isEmpty and total == 0
    - `testErrorResponseHandling`: Verify a 401 status code JSON can be detected

    Run full test suite to confirm.
