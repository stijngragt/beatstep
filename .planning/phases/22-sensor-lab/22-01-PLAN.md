    Create SensorLabService in BeatStep/Services/ following the CadenceService singleton pattern:
    - `@Observable final class SensorLabService` with `static let shared`
    - Observable properties: accelerationX, accelerationY, accelerationZ (Double), stepCount (Int), isRunning (Bool), detectionInterval (TimeInterval, default 1.0)
    - `@ObservationIgnored private var motionManager: CMMotionManager?`
    - Public `var samples: [AccelerometerSample] = []` (needs to be readable by chart view)
    - `private let maxSamples = 100`
    - `startAccelerometer()`: lazy-init CMMotionManager, set accelerometerUpdateInterval, startAccelerometerUpdates on OperationQueue.main, set isRunning=true
    - `stopAccelerometer()`: stopAccelerometerUpdates, set motionManager=nil, isRunning=false, clear samples
    - `updateInterval(_ newInterval: TimeInterval)`: set detectionInterval, if isRunning then stop+start to apply new interval (CMMotionManager reads interval at start time per research pitfall 4)
    - `appendSample(_ data: CMAccelerometerData)` or internal method: create AccelerometerSample, append, trim to maxSamples
    - Mark appendSample and samples manipulation as internal (not private) so tests can call appendSample directly without requiring real hardware

    - Do NOT test startAccelerometer/stopAccelerometer with real hardware -- only test the pure logic
