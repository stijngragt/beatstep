<objective>
Tune cadence display and song selection responsiveness so runners feel the app is tracking them in real time.

Purpose: The current 5s rolling window and 17s debounce make the app feel sluggish. Runners notice a delay between changing pace and seeing the number update or hearing a new song. This plan tightens both loops and adds a dead zone filter to prevent jitter.

Output: Modified CadenceService (2.5s window + dead zone filter), modified RunEngineService (8s debounce), updated and new tests proving all three CAD requirements.
</objective>

private func onCadenceChanged(_ newSPM: Int) {
    let significantChange = abs(newSPM - sustainedSPM) > tolerance.range
    guard significantChange else {
        sustainedChangeTask?.cancel()
        sustainedChangeTask = nil
        return
    }
    sustainedChangeTask?.cancel()
    sustainedChangeTask = Task { [weak self] in
        try? await Task.sleep(for: .seconds(17))  // TARGET: change to 8
        guard !Task.isCancelled else { return }
        self?.sustainedSPM = newSPM
        self?.invalidateBuffer()
    }
}
