    **1. Info.plist + project.yml -- UIUserInterfaceStyle = Dark (DARK-01 primary mechanism):**
    Add `UIUserInterfaceStyle: Dark` to the `info.properties` section in project.yml (under the BeatStep target, alongside existing properties like LSApplicationQueriesSchemes). This ensures the key persists across XcodeGen regeneration. Also add the key directly to Info.plist for immediate effect:
    ```xml

    ```
    Add it inside the top-level `<dict>`, after the existing keys.

    **2. BeatStepApp.swift -- window-level dark override (DARK-01 belt-and-suspenders):**
    Add `.onAppear` to the ContentView() inside WindowGroup that iterates all connected scenes/windows and sets `overrideUserInterfaceStyle = .dark`. This covers system-presented UI (alerts, sheets, Spotify OAuth webview) that Info.plist alone may not propagate to:
    ```swift
    ContentView()
        .environment(authService)
        .onAppear {
            for scene in UIApplication.shared.connectedScenes {
                guard let windowScene = scene as? UIWindowScene else { continue }
                for window in windowScene.windows {
                    window.overrideUserInterfaceStyle = .dark
                }
            }
        }
    ```

    **3. RunView.swift -- remove per-view overrides (DARK-02):**
