    - SpotifyAPIService.searchTrack returns matching Spotify tracks
    - SpotifyAPIService.createPlaylist creates private playlist on user account

  </behavior>
  <action>
    1. **SpotifyAPIService.swift** -- Extend with POST support and new endpoints:
       - Change `authenticatedRequest` from private to internal access level.
       - Add `authenticatedPOSTRequest<T: Decodable>(url: URL, body: Encodable) async throws -> T` -- same pattern as GET but with HTTPMethod POST, Content-Type application/json, and encoded body.
       - Add `searchTrack(title: String, artist: String, limit: Int = 1) async throws -> [SpotifyTrack]` -- GET `/search?q=track:{title}+artist:{artist}&type=track&limit={limit}`. Decode nested `tracks.items`. Use precise query format per RESEARCH.md Pitfall 4 (Spotify search limit=10).
       - Add `fetchCurrentUserID() async throws -> String` -- calls fetchCurrentUserProfile(), returns id.
       - Add `createPlaylist(userID: String, name: String, description: String) async throws -> SpotifyPlaylist` -- POST to `/users/{userID}/playlists` with body `{"name": name, "description": description, "public": false}`.
       - Add `addTracksToPlaylist(playlistID: String, uris: [String]) async throws` -- POST to `/playlists/{playlistID}/tracks` with body `{"uris": uris}`. Response can be discarded (returns snapshot_id).

    2. **LibraryScanService.swift** -- `@Observable` singleton following RESEARCH.md Pattern 4:
       - Published `scanProgress: ScanProgress?` (struct with playlistName: String, scanned: Int, total: Int, computed isComplete).
       - `func scanPlaylist(_ playlist: SpotifyPlaylist, tracks: [SpotifyTrack]) async`:
         a. Filter tracks to only those where `BPMCacheService.shared.hasLookup(forTrackID:)` returns false (delta scan).
         b. If all cached, update ScannedPlaylist coverage and return.
         c. Set scanProgress with total = uncached count.
         d. For each uncached track: call `GetSongBPMService.shared.fetchBPM(title:artist:)` in a do/catch (skip on error), cache result via `BPMCacheService.shared.cache(...)`, increment scanned.
         e. After completion: update the ScannedPlaylist's tracksWithBPM and lastScanned. Set scanProgress to nil.
       - `func scanEnabledPlaylists() async`: Fetch all ScannedPlaylist where isEnabled, for each load tracks from SpotifyAPIService and call scanPlaylist. This is the "on app launch delta scan".

    3. **BPMDiscoveryService.swift** -- Singleton for SPOT-05 discovery capability:
       - Private `discoveryPlaylistID: String?` (cached after first creation).
       - `func discoverTracks(atBPM bpm: Int) async throws -> [SpotifyTrack]`:
         a. Call `GetSongBPMService.shared.fetchSongsByBPM(bpm)` to get song list.
         b. For each result (limit first 10), search Spotify catalog via `SpotifyAPIService.shared.searchTrack(title:artist:)`.
         c. Collect matched SpotifyTrack results, return them.
         d. Add 300ms delay between Spotify searches for rate limiting.
       - `func saveToDiscoveryPlaylist(tracks: [SpotifyTrack]) async throws`:
         a. If discoveryPlaylistID is nil, create "BeatStep Discoveries" playlist silently via SpotifyAPIService.
         b. Add track URIs to playlist via addTracksToPlaylist.
       - Store discoveryPlaylistID in UserDefaults for persistence across launches.

    4. **LibraryScanServiceTests.swift** -- Unit tests with mocked dependencies:
       - Test that scanPlaylist skips already-cached tracks (delta scan logic).
       - Test that scanProgress updates correctly (starts at 0/N, ends at N/N, then nil).
       - Test that failed BPM lookups are cached with nil bpm (not skipped entirely).
       - Use in-memory SwiftData container. Mock GetSongBPMService by making it accept a custom URLSession with URLProtocol mock (or make the service protocol-based for testing).

    **Important considerations:**
    - SpotifyAPIService POST support needs a new struct for playlist creation body (make it private/internal Codable struct).
    - For the POST response discard pattern (addTracksToPlaylist), create a minimal `SnapshotResponse: Codable` with `snapshotId: String` or use a non-generic POST method that doesn't decode.
    - LibraryScanService needs to access ScannedPlaylist SwiftData models -- use BPMCacheService's context or accept ModelContext as parameter.
