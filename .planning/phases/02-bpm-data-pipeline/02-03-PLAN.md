**1. Create the Cloudflare Worker proxy** in `bpm-proxy/` directory at project root:

`bpm-proxy/package.json`: Minimal package with `wrangler` as devDependency and a `deploy` script.

`bpm-proxy/wrangler.toml`: Worker config with name `bpm-proxy`, compatibility_date of today, main pointing to `src/index.ts`. The `GETSONGBPM_API_KEY` is a secret (set via `wrangler secret put`, NOT in toml).

`bpm-proxy/src/index.ts`: A Cloudflare Worker (~30 lines) that:
- Accepts GET requests at any path
- Reads the incoming URL path and query params
- Strips the `api_key` param from the original request (if present)
- Forwards the request to `https://api.getsongbpm.com` with the same path, injecting `api_key` from `env.GETSONGBPM_API_KEY`
- Returns the API response with CORS headers (`Access-Control-Allow-Origin: *`)
- Returns 405 for non-GET methods
- Returns 500 with error message on fetch failures

Example: iOS app requests `https://bpm-proxy.{user}.workers.dev/search/?type=song&lookup=Run+Boy+Run` and the worker forwards to `https://api.getsongbpm.com/search/?api_key=SECRET&type=song&lookup=Run+Boy+Run`.

**2. Update GetSongBPMService.swift:**
- Add a new property `proxyBaseURL` that reads from `Secrets.getSongBPMProxyURL` (the deployed worker URL, e.g., `https://bpm-proxy.username.workers.dev`)
- Change `fetchBPM` to use `proxyBaseURL` instead of `baseURL` for requests. Remove the `api_key` query parameter from requests (the proxy injects it server-side). Remove the User-Agent and Accept spoofing headers (the proxy handles the server-side request so these are no longer needed).
- Change `fetchSongsByBPM` similarly to use `proxyBaseURL` without `api_key`.
- Keep `baseURL` and `apiKey` as fallback properties but the primary path uses the proxy.

**3. Update Secrets.example.swift:**
- Add `static let getSongBPMProxyURL = "YOUR_CLOUDFLARE_WORKER_URL"` (e.g., `https://bpm-proxy.username.workers.dev`)

**1. Rewrite LibraryScanService.scanPlaylist** to use GetSongBPMService instead of SpotifyAPIService.fetchBatchAudioFeatures:

Replace the batch audio-features approach (lines 38-59) with a per-track GetSongBPM lookup:

```swift
func scanPlaylist(_ playlist: SpotifyPlaylist, tracks: [SpotifyTrack]) async {
    let uncachedTracks = tracks.filter { !BPMCacheService.shared.hasLookup(forTrackID: $0.id) }

    updateScannedPlaylist(playlistID: playlist.id, name: playlist.name, totalTracks: tracks.count)

    if uncachedTracks.isEmpty { return }

    scanProgress = ScanProgress(playlistName: playlist.name, scanned: 0, total: uncachedTracks.count)

    for track in uncachedTracks {
        do {
            let bpm = try await GetSongBPMService.shared.fetchBPM(
                title: track.name,
                artist: track.artistName
            )
            BPMCacheService.shared.cache(
                trackID: track.id,
                name: track.name,
                artist: track.artistName,
                bpm: bpm
            )
        } catch {
            // Network/API error for this track -- mark as attempted with nil
            // so delta scan doesn't retry immediately
            BPMCacheService.shared.cache(
                trackID: track.id,
                name: track.name,
                artist: track.artistName,
                bpm: nil
            )
        }
        scanProgress?.scanned += 1
    }

    // Update coverage stats
    let allTrackIDs = tracks.map(\.id)
    let stats = BPMCacheService.shared.coverageStats(forTrackIDs: allTrackIDs)
    updateScannedPlaylistCoverage(playlistID: playlist.id, tracksWithBPM: stats.withBPM)

    scanProgress = nil
}
```

- Remove all `SpotifyAPIService.shared.fetchBatchAudioFeatures` references

- Rate limiting is handled inside GetSongBPMService (300ms delay between search and detail calls)
- Error handling per-track (not per-batch) -- a single track failure doesn't lose the whole batch

**2. Remove the `fetchBatchAudioFeatures` import/usage.** LibraryScanService should have no reference to SpotifyAPIService for BPM data. SpotifyAPIService is still used in `scanEnabledPlaylists` for fetching playlist tracks -- that's correct and stays.
