## Component Inventory

### Expanded Player (existing: `MiniPlayerView`)

Current layout preserved. Height: `ComponentSize.miniPlayerHeight` (64pt).

```
[BPM Badge 52x44] [Track Name / Artist] [Spacer] [Play/Pause] [Skip]
```

- Background: `.ultraThinMaterial` with top shadow
- Horizontal padding: `Spacing.md` (16pt)
- Vertical padding: `Spacing.sm` (8pt)
- HStack spacing: `Spacing.md` (16pt)
- Control spacing: `Spacing.lg` (24pt)

### Collapsed Player (new)

```
[                    ---- pill ----                    ]
```

- Total height: 20pt (D-06)
- Background: `.ultraThinMaterial` full-width (same as expanded, D-06)
- Top shadow: same as expanded (D-07)
- Pill handle: centered capsule, 36pt wide x 4pt tall (D-05)
- Pill color: `Color.textTertiary` (white 35%) -- subtle but visible against material
- Hit target: 44pt tall invisible overlay centered on pill for tap-to-expand

### Transition Between States (D-08, D-09)

During interactive drag:
1. Height interpolates between 64pt (expanded) and 20pt (collapsed)
2. Content opacity: `1.0` at expanded, `0.0` at collapsed, linearly tied to drag progress
3. Pill opacity: `0.0` at expanded, `1.0` at collapsed, inverse of content opacity (D-09 cross-fade)
4. Both content and pill are always in the view tree; only opacity changes

## Interaction Contract

### Gesture: Interactive Drag (D-03)

| Property | Value |
|----------|-------|
| Gesture type | `DragGesture(minimumDistance: 8)` on the player container |
| Direction to collapse | Swipe down (positive Y translation) |
| Direction to expand | Swipe up (negative Y translation) on collapsed handle |
| Tracking | Player height follows finger position during drag |
| Threshold to snap | 40pt of drag distance (Claude's discretion) |
| Below threshold | Snap back to original state |
| Above threshold | Snap to opposite state |

### Snap Animation

| Property | Value | Source |
|----------|-------|--------|
| Animation | `BSAnimation.smooth` | `.spring(response: 0.45, dampingFraction: 0.85)` |
| Haptic on snap | `BSHaptics.light()` | D-04 decision |
| Haptic timing | Fires once when drag crosses threshold, not on release |

### Tap Shortcuts (D-01, D-02)

| State | Tap Action | Target |
|-------|------------|--------|
| Expanded | Tap anywhere on player background (not on controls) to collapse | Toggle to collapsed |
| Collapsed | Tap anywhere on collapsed bar to expand | Toggle to expanded |

Note: In expanded state, tap-to-collapse must NOT conflict with play/pause and skip button taps. Use gesture priority: buttons get `.highPriorityGesture`, background tap is lower priority, or use a tap-only overlay excluding button areas.

### State Persistence (D-10, D-11)

| Property | Value |
|----------|-------|
| Storage | `@AppStorage("playerCollapsed")` |
| Type | `Bool` |
| Default | `false` (expanded on fresh install, D-10) |
| Persists across | App restarts, track changes, tab switches |

## Safe Area Contract

The `miniPlayerInset` ViewBuilder in `ContentView.swift` must change height based on collapsed state:

| State | Inset Height | Rationale |
|-------|-------------|-----------|
| Expanded | 64pt (`ComponentSize.miniPlayerHeight`) | Current behavior |
| Collapsed | 20pt (`ComponentSize.miniPlayerCollapsedHeight`) | Content scrolls closer to tab bar |
| No track | 0pt | Current behavior (player hidden) |

The `safeAreaInset` value must animate with `BSAnimation.smooth` when toggling states.
