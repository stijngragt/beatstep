  <name>Task 1: Create RunView and CadenceDisplayView</name>
  <files>BeatStep/Views/Run/RunView.swift, BeatStep/Views/Run/CadenceDisplayView.swift</files>
  <action>
Create `BeatStep/Views/Run/CadenceDisplayView.swift`:
- Takes `spm: Int` and `trend: CadenceTrend` as parameters
- Hero SPM display: large monospaced font (system size ~72-80, bold), white color
- "SPM" label below the number in smaller font, secondary color
- Trend arrow next to or below SPM number:
  - `.speedingUp`: SF Symbol `arrow.up` in green
  - `.steady`: SF Symbol `arrow.right` in white/gray
  - `.slowingDown`: SF Symbol `arrow.down` in red/orange
- Trend arrow font size ~24 to be visible at a glance

Create `BeatStep/Views/Run/RunView.swift`:
- Init parameter: `let playlist: SpotifyPlaylist`
- Private computed: `var cadenceService: CadenceService { .shared }`
- Dark background: `Color.black.ignoresSafeArea()` with `.preferredColorScheme(.dark)`
- Layout as ZStack with VStack content:

**State-driven main content (switch on cadenceService.state):**

`.idle` state (pre-run):
- Playlist name/context at top (small text, secondary color)
- "Ready to Run" message
- Large "Start Run" button (prominent, green or bright accent color)
- If `cadenceService.permissionDenied`: show permission denied view instead -- explanation text "BeatStep needs motion access to detect your running cadence", button "Open Settings" that calls `UIApplication.shared.open(URL(string: UIApplication.openSettingsURLString)!)`

`.detecting` state:
- Pulsing or animated "Detecting..." text (use opacity animation or ProgressView)
- Show that cadence is settling, approximately 5 seconds

`.active` state:
- `CadenceDisplayView(spm: cadenceService.currentSPM, trend: cadenceService.trend)` as hero element
- Playlist name in small text above or below

`.paused` state:
- "Paused" text in large font (dimmed)
- Subtitle: "Resume running to continue"
- Keep last known SPM visible but dimmed

**Controls:**
- When state is `.idle`: "Start Run" button calls `cadenceService.requestPermissionAndStart()` and sets `UIApplication.shared.isIdleTimerDisabled = true`
- When state is NOT `.idle`: "Stop Run" button (red, bottom of screen) calls `cadenceService.stopDetecting()` and sets `UIApplication.shared.isIdleTimerDisabled = false`

**Mini-player:** Embed `MiniPlayerView()` at the bottom of the ZStack, positioned above safe area

**Cleanup in `.onDisappear`:**
- Set `UIApplication.shared.isIdleTimerDisabled = false`
- Call `cadenceService.stopDetecting()` (safety net -- stops if user navigates away without tapping Stop)

Design notes per user decisions:
- Dark background with bright text for outdoor visibility
- Cadence is THE hero element -- large, centered, dominant
- Glanceable -- runner should understand state in <1 second glance
- No pause/resume concept -- just start and stop
- No summary screen on stop -- clean return
