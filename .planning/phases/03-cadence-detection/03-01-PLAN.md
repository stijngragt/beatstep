  <action>
Create `BeatStep/Models/RunSession.swift` with:
- `CadenceState` enum: `.idle`, `.detecting`, `.active`, `.paused`
- `CadenceTrend` enum: `.speedingUp`, `.steady`, `.slowingDown`

Create `BeatStepTests/Mocks/MockPedometerData.swift` with a helper that creates test data for the smoothing logic. Since CMPedometerData cannot be directly instantiated in tests, the CadenceService smoothing and trend logic should be testable via internal methods that accept raw cadence values (Double). Make `processCadenceSample(_ spm: Double, at timestamp: Date)` an internal method on CadenceService so tests can call it directly without needing a real CMPedometer.

Create `BeatStepTests/CadenceServiceTests.swift` with the tests described in behavior. Tests exercise the smoothing math, trend detection, and state transitions by calling `processCadenceSample` directly.

Create `BeatStep/Services/CadenceService.swift` as an `@Observable` singleton:
- `static let shared = CadenceService()`
- Published state: `currentSPM: Int`, `trend: CadenceTrend`, `state: CadenceState`, `permissionDenied: Bool`
- Private: `pedometer: CMPedometer`, `cadenceWindow: [(timestamp: Date, cadence: Double)]`, `windowDuration: TimeInterval = 5.0`, `lastStepTime: Date?`, `trendHistory: [Double]`, `inactivityTimer: Timer?`
- `requestPermissionAndStart()`: Check `CMPedometer.authorizationStatus()`, handle `.denied`/`.restricted` by setting `permissionDenied = true`, otherwise call `startDetecting()`
- `startDetecting()`: Guard `CMPedometer.isCadenceAvailable()`, set state to `.detecting`, call `pedometer.startUpdates(from: Date())`, start inactivity monitor timer
- `stopDetecting()`: Call `pedometer.stopUpdates()`, invalidate inactivity timer, reset all state to defaults
- `processCadenceSample(_ spm: Double, at timestamp: Date)` (internal access): Add to window, prune old samples, compute rolling average, update `currentSPM`, transition from `.detecting` to `.active`, call `updateTrend`, update `lastStepTime`, reset state from `.paused` to `.active` if was paused
- `updateTrend(currentAvg: Double)` (private): Maintain `trendHistory` of last 5 samples, compute delta between first and last, apply 5 SPM threshold for trend change
- Pedometer handler in `startDetecting` dispatches to `@MainActor` / `DispatchQueue.main.async`, extracts `currentCadence`, multiplies by 60, calls `processCadenceSample`. Include fallback: if `currentCadence` is nil but `numberOfSteps` changed, compute cadence from step delta / time delta.
- Inactivity monitor: Timer every 2 seconds checks if `lastStepTime` is >5 seconds ago while state is `.active`, transitions to `.paused`

Follow project patterns: `@Observable`, `private init()`, singleton `.shared`.
