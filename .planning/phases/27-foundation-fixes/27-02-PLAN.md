1. In PlaylistListView, add `.onChange(of:)` modifier to reactively reload coverage data when a scan completes:
   - Add to the `playlistList` view (on the List or inside the body):
   ```swift
   .onChange(of: scanService.scanningPlaylistID) { oldValue, newValue in
       if oldValue != nil && newValue == nil {
           // Scan just completed, reload coverage data
           loadCoverageData()
       }
   }
   ```

2. This ensures that when `scanningPlaylistID` transitions from a playlist ID to nil (scan complete), the coverage data is immediately refreshed from SwiftData.

3. The existing swipe action Task that calls `loadCoverageData()` after `scanPlaylistByID` can remain -- it provides a safety net, but the `.onChange` is the reactive trigger that actually fires reliably.
