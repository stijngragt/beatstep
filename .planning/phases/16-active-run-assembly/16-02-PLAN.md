  <action>
**RunView.swift modifications:**

1. Add @State property: `@State private var showActiveRun = false`

2. Add .fullScreenCover modifier to the outer ZStack:
   ```swift
   .fullScreenCover(isPresented: $showActiveRun) {
       ActiveRunView(playlist: playlist, tracks: tracks, selectedZoneId: selectedZoneId)
           .interactiveDismissDisabled(true)
   }
   ```

3. Trigger fullScreenCover: Add `.onChange(of: cadenceService.state)` to detect when state transitions to `.active`. When cadenceService.state becomes `.active`, set `showActiveRun = true`. Use:
   ```swift
   .onChange(of: cadenceService.state) { oldValue, newValue in
       if newValue == .active && oldValue != .active {
           showActiveRun = true
       }
   }
   ```
   Note: CadenceState must be Equatable for onChange to work. If it is not already, this needs to be added. Check before implementing.

4. Guard onDisappear (Research Pitfall 3): Change the existing `onDisappear` to only clean up if the run is NOT active:
   ```swift
   .onDisappear {
       if !runEngine.isRunActive {
           runEngine.stopRun()
           UIApplication.shared.isIdleTimerDisabled = false
           cadenceService.stopDetecting()
       }
   }
   ```
   This prevents the run from stopping when fullScreenCover presents (which may trigger onDisappear on the presenting view).

5. When showActiveRun becomes false (fullScreenCover dismissed by ActiveRunView's stop action), the run is already stopped (ActiveRunView handles cleanup in its onStop). No additional cleanup needed in RunView -- the onDisappear guard handles the rest if the user navigates away.

6. Keep the existing activeView case in mainContent for the brief moment between state becoming .active and fullScreenCover appearing (one frame). The fullScreenCover will immediately cover it.

7. Also handle the case where fullScreenCover dismisses and cadence might still be active (edge case): when `showActiveRun` changes to false, check if cadence is still going and handle accordingly.

**ContentView.swift modifications:**

8. Update the MiniPlayer safeAreaInset condition to also check isRunActive:
   ```swift
   .safeAreaInset(edge: .bottom) {
       if SpotifyPlayerService.shared.currentTrack != nil && !RunEngineService.shared.isRunActive {
           MiniPlayerView()
       }
   }
   ```
   This hides MiniPlayer when the active run is showing. Per Research Pitfall 4, set isRunActive before presenting fullScreenCover (already handled -- RunEngineService.startRun sets isRunActive = true, which happens before cadence becomes active).

Tab bar hiding is automatic with fullScreenCover -- no code needed.
