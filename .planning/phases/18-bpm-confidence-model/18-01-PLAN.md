- Add convenience accessors: `var isManual: Bool { confidence == .manual }` and `var isVerified: Bool { confidence == .verified }`
- Do NOT add confidenceRaw/sourceRaw to the init -- they remain nil by default for new records (set explicitly by write paths)

    - cacheFromAPI overwrites BPM when existing record has non-manual confidence
    - cacheManual takes non-optional bpm: Int and always sets .manual confidence and .manual source
    - cacheManual overwrites any existing BPM regardless of current confidence

In `BeatStep/Services/BPMCacheService.swift`:

1. Remove the existing `cache(trackID:name:artist:bpm:)` method entirely.

2. Add `cacheFromAPI(trackID: String, name: String, artist: String, bpm: Int?)`:
   - Fetch existing CachedBPM by trackID using FetchDescriptor with #Predicate
   - If existing record found:
     - Check `existing.confidenceRaw == BPMConfidence.manual.rawValue` -- if true, only update `lookupAttempted = true` and `lastUpdated = Date()` (preserve manual BPM, do NOT overwrite bpm/confidence/source)
     - Otherwise: update `existing.bpm = bpm`, set `confidenceRaw = bpm != nil ? BPMConfidence.verified.rawValue : nil`, set `sourceRaw = bpm != nil ? BPMSource.api.rawValue : nil`, update `lookupAttempted = true`, `lastUpdated = Date()`
   - If no existing record: create new CachedBPM, set confidenceRaw/sourceRaw only if bpm is non-nil, insert into context
   - Save context

3. Add `cacheManual(trackID: String, name: String, artist: String, bpm: Int)` (note: bpm is non-optional per user decision):
   - Fetch existing CachedBPM by trackID
   - If existing: update bpm, set `confidenceRaw = BPMConfidence.manual.rawValue`, `sourceRaw = BPMSource.manual.rawValue`, `lookupAttempted = true`, `lastUpdated = Date()`
   - If no existing: create new CachedBPM with bpm, set confidenceRaw and sourceRaw to manual, insert
   - Save context

IMPORTANT: Use `confidenceRaw` (String?) directly in write paths and predicates, never the computed `confidence` property. This avoids triggering lazy backfill defaults during writes (per Pitfall 4 in research).
