```swift
private func updateScannedPlaylist(playlistID: String, name: String, totalTracks: Int) {
    let context = BPMCacheService.shared.context
    let descriptor = FetchDescriptor<ScannedPlaylist>(
        predicate: #Predicate { $0.spotifyPlaylistID == playlistID }
    )
    if let existing = try? context.fetch(descriptor).first {
        existing.totalTracks = totalTracks
        existing.lastScanned = Date()
    } else {
        let newRecord = ScannedPlaylist(
            spotifyPlaylistID: playlistID,
            name: name,
            totalTracks: totalTracks
        )
        newRecord.lastScanned = Date()
        context.insert(newRecord)
    }
    try? context.save()
}
```

```swift
.onChange(of: scanService.scanCompletionCount) { _, _ in

```

Keep the existing `.onChange(of: scanService.scanningPlaylistID)` observer as-is (per D-01, D-02). The scanningPlaylistID observer handles mid-scan UI updates (e.g., clearing scan progress), while scanCompletionCount handles the data refresh. Both serve different purposes.

**Why this works:** `scanCompletionCount` is an `@Observable` property on LibraryScanService. SwiftUI's observation system tracks access to it in the view's body (via `.onChange`). When `scanPlaylistByID` increments it, SwiftUI triggers the `.onChange` closure, which calls `loadCoverageData()` to re-fetch all ScannedPlaylist records from SwiftData. This covers:
- Manual scans from swipe actions
- Manual scans from context menu
- Background scans from `scanEnabledPlaylists()` at app launch
