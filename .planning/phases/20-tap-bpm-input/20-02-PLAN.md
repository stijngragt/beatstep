  <action>
    **Create TapBPMView.swift** as a SwiftUI half-sheet with these sections (top to bottom):

    1. **Header section:** Track name (.heading font, .textPrimary), artist name (.captionText, .textSecondary), live BPM display showing engine.currentBPM as large text or "--" when nil, tap count as "N/8" where N = max(0, engine.tapCount - 1) capped at 8.

    2. **Progress dots:** HStack of 8 Circle views. Filled (Color.accent) for index < completedIntervals, hollow (Color.surfaceOverlay) otherwise. completedIntervals = max(0, engine.tapCount - 1) capped at 8. When engine.isStable, show HStack with checkmark icon + "Stable" text in .stateSuccess color.

    3. **Tap zone:** Rectangle filling available space with Color.surfaceOverlay fill. Overlay with hand.tap.fill icon (size 48) + "Tap along with the beat" text. Use .contentShape(Rectangle()) + .onTapGesture that calls engine.tap(). After tap, check engine.lastTapWasOutlier:
       - If outlier: UINotificationFeedbackGenerator .error, trigger shake animation via ShakeModifier
       - If valid: UIImpactFeedbackGenerator .light, brief flash animation (opacity pulse)

    4. **Bottom bar:** HStack with Reset button (left, calls engine.reset()) and Save button (right, calls save()). Save disabled when !engine.canSave. Save button styled with Color.accent background, white text.

    5. **save() function:** Guard engine.currentBPM, call BPMCacheService.shared.cacheManual(trackID:name:artist:bpm:), fire UINotificationFeedbackGenerator .success, call onSave(bpm), dismiss().

    6. **onAppear:** Call SpotifyPlayerService.shared.play(uri: track.uri, contextURI: playlistURI).
    7. **onDisappear:** Call engine.reset() to invalidate timers.

    8. **ShakeModifier:** ViewModifier that offsets x by -6 when animating, with .default.repeatCount(3, autoreverses: true).speed(6) animation. (See research code example.)

    Init parameters: track: SpotifyTrack, playlistURI: String, onSave: (Int) -> Void.
    State: @State engine = TapBPMEngine(), @State showTapFlash = false, @State showShake = false, @Environment(\.dismiss) dismiss.

    **Modify PlaylistDetailView.swift:**

    1. Add `@State private var tapBPMTrack: SpotifyTrack?` property.

    2. Add `.sheet(item: $tapBPMTrack)` modifier on the trackList or Group, presenting TapBPMView with:
       - track: the bound track
       - playlistURI: "spotify:playlist:\(playlist.id)"
       - onSave: { _ in bpmCache[track.id] = BPMCacheService.shared.getBPMInfo(forTrackID: track.id) }
       - .presentationDetents([.medium])

    3. Modify TrackRow: Add `var onBadgeTap: (() -> Void)? = nil` parameter. Wrap the BPM badge (both the if-let branch and the else branch) in a Button that calls onBadgeTap?() with .buttonStyle(.plain). This makes the badge capture taps before the row's .onTapGesture.

    4. At the TrackRow call site (line ~71), pass onBadgeTap: { tapBPMTrack = track }.

    Per user decisions: Button approach for badge prevents gesture conflict with row tap (row tap plays track, badge tap opens sheet). Use .buttonStyle(.plain) to preserve visual styling.
