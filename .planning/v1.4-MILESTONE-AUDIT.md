## Gap Detail

### SLAB-02 (Partial): stepCount always 0

**Location:** `SensorLabService.swift` line 13 (declared), line 51 (reset to 0), never incremented.
**Displayed at:** `SensorLabView.swift` line 34 — always shows "0 steps".
**Root cause:** The actual step count lives in `CadenceService`'s private `previousStepCount`, which is not publicly exposed. `SensorLabService` declares `stepCount` but has no data source for it.
**Fix options:**
1. Expose cumulative steps from `CadenceService` and bind SensorLabView to it
2. Add CMPedometer query to SensorLabService for independent step counting
3. Remove stepCount display from SensorLabView (reduce scope)
