  <name>Task 2: Implement SpotifyAuthService, KeychainManager, and LoginView</name>
  <files>

  <action>
    **Create `KeychainManager.swift`:**
    - Singleton with `static let shared`
    - Uses KeychainAccess library with service identifier "com.beatstep.app"
    - Properties: `accessToken: String?` (get/set from Keychain), `tokenExpirationDate: Date?`
    - Method: `clearAll()` -- removes all stored tokens
    - All get/set operations use `keychain["access_token"]` and `keychain["token_expiration"]`

    **Create `SpotifyAuthService.swift`:**
    - `@Observable` class, singleton `static let shared`
    - Published state: `isAuthenticated: Bool`, `isCheckingAuth: Bool`, `isPremium: Bool`, `currentUser: SpotifyUser?`, `authError: String?`
    - Private: `clientID` (use placeholder "YOUR_SPOTIFY_CLIENT_ID" with a comment to replace), `redirectURL = URL(string: "beatstep://spotify-callback")!`
    - Lazy `appRemote: SPTAppRemote` configured with clientID and redirectURL
    - `func initiateAuth()`:
      - Check if Spotify is installed via `UIApplication.shared.canOpenURL(URL(string: "spotify:")!)`
      - If not installed, set authError to "Please install Spotify to use BeatStep" and return
      - Set `appRemote.connectionParameters.accessToken = ""` (triggers auth)
      - Call `appRemote.authorizeAndPlayURI("")` with scopes: `SPTAppRemoteScope` values for app-remote-control, playlist-read-private, playlist-read-collaborative, user-read-private, user-read-playback-state, user-read-currently-playing
      - Note: The SPTAppRemote iOS SDK uses `SPTScope` enum values. Use `appRemote.authorizationParameters` to pass scopes.
    - `func handleCallback(url: URL)`:
      - Extract access token from URL parameters using `SPTAppRemote.authorizationParameters(from: url)`
      - Store token in `KeychainManager.shared.accessToken`
      - Set `appRemote.connectionParameters.accessToken` to the token
      - Call `checkPremiumStatus()` async
    - `func checkPremiumStatus() async`:
      - Fetch user profile via `GET /v1/me` using the stored token (inline URLSession call or call SpotifyAPIService if available)
      - Check `user.product == "premium"`
      - If not premium: set `isPremium = false`, `authError = "BeatStep requires Spotify Premium"`, clear tokens
      - If premium: set `isPremium = true`, `isAuthenticated = true`, `currentUser = user`
    - `func disconnect()`:
      - Clear KeychainManager tokens
      - Reset all state (isAuthenticated = false, currentUser = nil, etc.)
    - `func checkExistingAuth()`:
      - On app launch, check if Keychain has a valid token
      - If yes, attempt to verify by calling `/v1/me`
      - If token works: set isAuthenticated = true, load user
      - If token expired/invalid: clear tokens, show login

    **Create `LoginView.swift`:**
    - SwiftUI view with BeatStep branding (app name, brief tagline "Your music, your stride")
    - "Connect with Spotify" button (green, Spotify brand color #1DB954)
    - Calls `SpotifyAuthService.shared.initiateAuth()` on tap
    - Shows loading state while auth is in progress
    - Shows error message if `authService.authError` is set
    - If premium check fails, show "BeatStep requires Spotify Premium" with options: "Try Different Account" (re-initiates auth) and "Learn More" (optional, links to Spotify premium)
    - If Spotify not installed, show "Install Spotify" button that opens App Store link

    - Playlist list response (2-3 playlists with typical fields)
    - Playlist tracks response (3-4 tracks with nested artist/album)
    - Error response (401 unauthorized)

    `SpotifyAuthServiceTests.swift`: XCTest class with:

    - `testKeychainClearAll`: Store token, clear, verify nil
    - `testPremiumUserParsing`: Decode premium user JSON fixture, verify `product == "premium"`
    - `testFreeUserParsing`: Decode free user JSON fixture, verify `product == "free"`
    - `testSpotifyErrorCases`: Verify all SpotifyError enum cases exist

    Run tests: `xcodebuild test -scheme BeatStep -destination 'platform=iOS Simulator,name=iPhone 16' -only-testing BeatStepTests`
