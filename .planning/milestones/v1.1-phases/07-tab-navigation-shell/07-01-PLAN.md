**1. Create `BeatStep/Views/Run/RunTabView.swift`:**

New file. Run tab root view with two states:

- **Idle state** (when `RunEngineService.shared.isRunActive == false`): Dark background (`Color.surfaceBase.ignoresSafeArea()`), centered vertically. A "Start Run" button styled as an accent-filled capsule: `Text("Start Run")` with `.font(.title3.weight(.bold))`, `.foregroundStyle(Color.textOnAccent)`, horizontal padding `Spacing.xl`, vertical padding `Spacing.md`, background `Capsule().fill(Color.accent)`. The button does nothing for now (Phase 8 NAV-04 adds playlist context and navigation). Below the button, add subtle hint text: `Text("Select a playlist from Library to start")` with `.font(.captionText)`, `.foregroundStyle(Color.textSecondary)`, `.padding(.top, Spacing.sm)`.

- **Active state** (when `RunEngineService.shared.isRunActive == true`): This state is entered via the existing Library > Playlist Detail > "Run with this Playlist" NavigationLink flow, NOT from the Run tab. The Run tab simply shows the idle CTA. The active RunView is pushed within the Library tab's NavigationStack, not the Run tab. So RunTabView only ever shows the idle CTA prompt.

Add `.navigationTitle("Run")` and `.navigationBarTitleDisplayMode(.large)`.

Note: RunEngineService does NOT have `currentPlaylist`/`currentTracks` properties, and RunView requires both as init parameters. The Run tab cannot instantiate RunView without them. The Start Run button is a placeholder -- Phase 8 (NAV-04) will add playlist context and the actual start flow.

**2. Restructure `BeatStep/App/ContentView.swift`:**

Replace the current single-NavigationStack + ZStack layout with a TabView architecture:

Add an `init()` method that configures UITabBarAppearance:
```swift
init() {
    let appearance = UITabBarAppearance()
    appearance.configureWithDefaultBackground()
    appearance.backgroundEffect = UIBlurEffect(style: .systemUltraThinMaterial)
    appearance.shadowColor = .clear  // No separator line (per user decision)
    UITabBar.appearance().standardAppearance = appearance
    UITabBar.appearance().scrollEdgeAppearance = appearance
    UITabBar.appearance().tintColor = UIColor(Color.accent)
    UITabBar.appearance().unselectedItemTintColor = UIColor(Color.textTertiary)
}
```

Key changes from current code:
- Remove the outer `ZStack(alignment: .bottom)` -- no longer needed
- Remove the `.toolbar { ToolbarItem }` with gear icon NavigationLink to Settings -- Settings is now its own tab
- Remove the `Color.clear.frame(height: 64)` safeAreaInset hack -- the new `.safeAreaInset(edge: .bottom) { MiniPlayerView() }` on TabView handles both space reservation AND rendering
- NavigationStack wraps EACH tab's content (not the TabView) -- this gives each tab independent nav state
- Keep `.onAppear` with AudioSessionService and SpotifyAuthService setup
- Keep the auth gate (if authenticated show TabView, else LoginView)

**SF Symbol icon notes** (per research): `music.note.list` and `waveform.path.ecg` do NOT have `.fill` variants. Only `gearshape`/`gearshape.fill` has a fill variant. SwiftUI automatically uses the fill variant when available for the selected tab. The tint color differentiates selected/unselected state for all icons.

**3. Remove MiniPlayerView from `BeatStep/Views/Run/RunView.swift`:**

Remove line 34 (`MiniPlayerView()`) from the VStack in RunView's body. The global safeAreaInset on the TabView now handles MiniPlayer rendering everywhere, including when RunView is active. Keeping it would cause double rendering.

Also remove the corresponding spacing/padding adjustments if the MiniPlayerView had any within that VStack.
