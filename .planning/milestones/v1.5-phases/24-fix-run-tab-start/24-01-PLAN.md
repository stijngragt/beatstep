  Rewrite RunTabView.swift to be the functional pre-run config screen. Accept `@Binding var selectedTab: Tab` init parameter.

  **State properties:**
  - `@Binding var selectedTab: Tab`
  - `@State private var playlist: SpotifyPlaylist?` (fetched playlist object)
  - `@State private var tracks: [SpotifyTrack]` = [] (fetched tracks)
  - `@State private var isLoading = false`
  - `@State private var loadError: String?`
  - `@State private var showActiveRun = false`
  - `@State private var selectedZoneId: Int?` = RunZone.selectedZoneId
  - `@State private var tolerance: BPMTolerance` = .saved
  - `@State private var lastFetchedPlaylistId: String?` (track which playlist ID was last fetched)
  - Computed: `private var cadenceService: CadenceService { .shared }` and `private var runEngine: RunEngineService { .shared }`

  **Eager fetch on .onAppear:**
  - If `LastRunPlaylist.id` is nil, show no-playlist state (skip fetch).
  - If `LastRunPlaylist.id != lastFetchedPlaylistId`, set `isLoading = true`, `loadError = nil`.
  - Call `SpotifyAPIService.shared.fetchPlaylist(id:)` and `SpotifyAPIService.shared.fetchPlaylistTracks(playlistID:)` concurrently using `async let`.
  - Extract `[SpotifyTrack]` from tracks response: `response.items.compactMap { $0.track }`.
  - On success: set `playlist`, `tracks`, `lastFetchedPlaylistId`, `isLoading = false`.
  - On failure: set `loadError` with a user-friendly message (e.g. "Couldn't load playlist"), `isLoading = false`. If error is 404 (playlist deleted/private), clear `LastRunPlaylist.name`, `LastRunPlaylist.id`, `LastRunPlaylist.imageURL` to trigger no-playlist state.
  - Also reload `selectedZoneId = RunZone.selectedZoneId` and `tolerance = .saved` on appear.

  **Body structure (three states):**

  *No-playlist state* (LastRunPlaylist.id == nil):
  - Center: "Pick a playlist to get started" text (.font(.captionText), .foregroundStyle(.textSecondary))
  - Below: "Go to Library" button that sets `selectedTab = .library`
  - Style "Go to Library" as a bordered button (use accent color border, not filled)
  - Start Run button at bottom, disabled with reduced opacity

  *Loading state* (isLoading == true):
  - Show ProgressView() centered with "Loading playlist..." text below it
  - Start Run button at bottom, disabled

  *Loaded state* (playlist != nil):
  - Same visual layout as current RunTabView (cover art, playlist name, "Your last playlist" subtitle, ZonePickerView, conditional TolerancePicker)
  - Make the playlist row (cover art + name area) tappable -- on tap, set `selectedTab = .library`
  - If `loadError != nil`: show error text and a "Retry" button (styled as borderless, accent-colored) below the playlist area

  **Start Run button (pinned at bottom, all states):**
  - Disabled when: `playlist == nil || tracks.isEmpty || isLoading`
  - On tap (copy logic from RunView.swift lines 213-228):
    1. Configure engine mode: if selectedZoneId maps to a zone, set `runEngine.runMode = .guided`, `runEngine.tolerance = tolerance`, `RunMode.savedTargetBPM = zone.bpm`. Else set `runEngine.runMode = .free`, `runEngine.tolerance = tolerance`.
    2. Call `cadenceService.requestPermissionAndStart()`
    3. Set `UIApplication.shared.isIdleTimerDisabled = true`
    4. `Task { await runEngine.startRun(playlist: playlist!, tracks: tracks) }`

  **fullScreenCover (same pattern as RunView.swift):**
  - `.fullScreenCover(isPresented: $showActiveRun)` presenting `ActiveRunView(playlist: playlist!, tracks: tracks, selectedZoneId: selectedZoneId).interactiveDismissDisabled(true)`
  - `.onChange(of: cadenceService.state)`: when state transitions to `.active`, set `showActiveRun = true`

  **On disappear cleanup (same as RunView):**
  - If `!runEngine.isRunActive`: call `runEngine.stopRun()`, disable idle timer, `cadenceService.stopDetecting()`

  **Zone onChange (keep existing):**
  - When selectedZoneId changes, save to `RunZone.selectedZoneId` and update RunMode as currently done.

  **Styling:** Keep all existing design tokens (Color.surfaceBase, Color.accent, Color.textPrimary, Spacing.*, Radius.*, ComponentSize.coverArtLarge, .font(.heading), .font(.captionText), etc.). The Start Run button style stays as-is (accent filled, full-width, rounded rectangle).
