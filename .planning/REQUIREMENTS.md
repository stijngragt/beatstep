# Requirements: BeatStep

**Defined:** 2026-03-25
**Core Value:** When you run, your music should move with you -- every footstrike landing on the beat.

## v1.5 Requirements

Requirements for v1.5 "One Way In". Each maps to roadmap phases.

### Run Flow

- [ ] **FLOW-01**: Starting a run always happens from the Run tab -- no other screen can initiate a run
- [x] **FLOW-02**: Run tab Start Run button reliably starts a run with the selected playlist, zone, and tolerance
- [ ] **FLOW-03**: Tapping "Run with this playlist" in Library navigates to Run tab with that playlist pre-loaded
- [ ] **FLOW-04**: The old playlist-initiated run screen (green button menu) is fully removed
- [x] **FLOW-05**: Returning users see last-used playlist, zone, and tolerance pre-loaded on Run tab -- one tap to start

### Onboarding

- [ ] **ONBD-01**: After Spotify and Health permissions, onboarding includes a step to analyze a first playlist before completion

## Future Requirements

### Run Flow Enhancements

- **FLOW-06**: Inline playlist picker in Run tab for swapping playlists without leaving the tab
- **FLOW-07**: Skip option in onboarding playlist analysis step

## Out of Scope

| Feature | Reason |
|---------|--------|
| Inline playlist picker in Run tab | Users go to Library to select playlists -- keeps Run tab simple |
| Multiple playlist queuing | One playlist per run is sufficient for v1.5 |
| Run history or stats | Not a fitness tracker -- focused on the live experience |
